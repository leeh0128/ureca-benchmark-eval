# VerilogEval — URECA Agentic Evaluation Harness

This repository is a fork of [NVlabs/verilog-eval](https://github.com/NVlabs/verilog-eval)
(MIT-licensed). It extends the upstream VerilogEval v2 spec-to-RTL benchmark with an
**agentic, feedback-driven evaluation harness** developed for the NTU URECA project
*"Towards AI for Secure Digital Circuit Design: An Open-Source Agentic Pipeline for
LLM-Driven RTL Generation."*

Modifications and extensions by Hyunseung Lee, Nanyang Technological University.
The upstream NVIDIA and OpenAI copyright notices are retained in `LICENSE`.

---

## What this fork adds

The stock VerilogEval harness scores a model in a single shot: it makes one LLM call
per problem and checks the generated RTL against a differential testbench. This fork
replaces that single-shot generator with an **iterative generate–test–fix loop** that
runs each candidate through an open-source EDA pipeline and feeds structured diagnostics
back to the model for correction.

The companion EDA pipeline (Verilator lint, Icarus Verilog simulation, Yosys synthesis,
OpenSTA timing/power, gate-level simulation) lives in a separate repository:

> **EDA pipeline:** https://github.com/leeh0128/ureca-verilog-agent.git

This repository contains only the benchmark-side integration: the drop-in generator,
the batch runner, and the result-aggregation scripts.

## Repository layout

```
veval_agentic_integration/   ← the agentic integration harness
  sv-generate-agentic        ← drop-in replacement for VerilogEval's sv-generate
  run-batch-eval.sh          ← runs a full 156-problem sweep for one configuration
  aggregate-tokens.py        ← collects per-problem token usage from .tokens.json sidecars
scripts/                     ← upstream + modified analysis scripts (e.g. sv-iv-analyze)
dataset_spec-to-rtl/         ← upstream VerilogEval v2 spec-to-RTL problem set
comparison.csv               ← three-way pass-rate / token / wall-time comparison
pass_rate_to_csv.py          ← extracts pass rates from build logs into CSV
count_failures.py            ← per-failure-code breakdown across configurations
```

> Per-run `build_spec-to-rtl_*` directories and result tarballs are intentionally
> excluded via `.gitignore`; they are regenerated by running the harness.

## How the benchmarking works

Three configurations are evaluated on the 156-problem VerilogEval v2 spec-to-RTL task,
all using the same model (Gemini 2.5 Flash-Lite, free tier, temperature 0, 0-shot) so
that differences reflect the feedback architecture rather than model capability:

1. **Single-shot baseline** — the stock `sv-generate` script: one LLM call per problem,
   no feedback, no iteration.
2. **Agentic with simulation feedback** — on each failed iteration, the Icarus Verilog
   mismatch output from the differential testbench is appended to the conversation and
   the model is asked to correct the RTL. Up to 5 iterations.
3. **Agentic with simulation + lint feedback** — identical to (2), but each failed
   candidate is additionally linted with Verilator, and the classified lint findings
   are appended alongside the simulation diagnostics.

The swap from single-shot to agentic generation is applied through a single environment
variable, so the upstream harness is otherwise untouched:

```bash
export GENERATE_VERILOG=sv-generate-agentic
```

`sv-generate-agentic` accepts the prompt file and `--output` path from the VerilogEval
Makefile, runs the full feedback loop internally, writes the final candidate as the
output `.sv` file, and logs per-problem token usage to a sidecar `.tokens.json`.

## Reproducing the results

> Prerequisites: a Gemini API key, and the EDA pipeline repository available so that
> `sv-generate-agentic` can invoke the lint/simulation stages. Run inside the
> IIC-OSIC-TOOLS container (or any environment with Verilator and Icarus Verilog).

```bash
# 1. Provide your API key (never commit this)
export GEMINI_API_KEY="your-key-here"

# 2. Configure the upstream harness (standard VerilogEval setup)
./configure
make

# 3. Run each configuration (example: agentic + lint)
export GENERATE_VERILOG=sv-generate-agentic
./veval_agentic_integration/run-batch-eval.sh

# 4. Aggregate results
python3 pass_rate_to_csv.py            # pass rates -> comparison.csv
python3 count_failures.py              # per-failure-code breakdown
python3 veval_agentic_integration/aggregate-tokens.py   # token usage
```

`comparison.csv` reproduces the three-way headline table reported in the paper
(pass rate, total tokens, average tokens/problem, wall time).
